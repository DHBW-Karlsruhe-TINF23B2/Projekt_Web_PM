ToDo Application

Los geht's

Unser Projekt ist eine minimalistische TODO-App, welche über einen Browser verwendet werden kann. Dazu haben wir unser Projekt in ein Backend und ein Frontend aufgeteilt. Dabei wurde für das Backend eine API-Schnittstelle programmiert. Das Frontend-Team hat eine Webseite, welche die API-Schnittstelle anspricht, erstellt. Unsere TODO-App soll einfach gehalten werden. Grundlegende Funktionen sind dabei die Darstellung aller erstellten TODOs, das Erstellen von neuen TODOs oder das Bearbeiten sowie das Löschen von vorhandenen TODOs. Ein TODO hat dabei einen Title, eine Beschreibung, die Information, ob das TODO bereits erledigt ist, sowie ein Datum wann dieses fällig ist. Dieses Fälligkeitsdatum soll nur dann angezeigt werden, wenn das TODO nicht erledigt ist.

Backend:

Das Backend stellt einen eigenständigen Service dar, welcher eine API-Schnittstelle bereitstellt. Über diese API können TODOs in der Datenbank angelegt, gelöscht, angepasst und abgerufen werden. Das Backend übernimmt dazu alle nötigen Aktionen und Validierungen der eingehenden Daten. Es teilt dem Sendenden mit, ob seine Aktion erfolgreich war oder was schiefgelaufen ist. Durch diese Schnittstelle ist es möglich Plattformunabhängig auf die Daten zuzugreifen. Diese werden im JSON Format übermittelt, sodass ein beliebiger Endpunkt zur Darstellung verwendet werden kann.

Frontend:

Das Frontend übernimmt die Aufgabe alle gespeicherten TODOs darzustellen, sowie eine benutzerfreundliche Möglichkeit zur Bearbeitung und Erstellung der TODOs anzubieten. Hierzu werden entsprechend dem AJAX-Modell die vom Backend übermittelten JSON Daten mit Javascript ausgewertet und als HTML-Code der Webseite dynamisch hinzugefügt. Hiermit gehen die typischen Verteile einer dynamischen Webseite einher, wodurch ein erneutes Laden der Webseite beim Hinzufügen, Ändern oder Löschen von TODOs nicht erforderlich ist. Darüber hinaus ist die Webseite responsive, sodass das Layout dynamisch auf die Größe des Browsers reagiert.
